# UC-020: Ver el estado de sus reports

**Actor**: Usuario que reportó
**Epic**: [EPIC-05: Sistema de reseñas](../epics/EPIC-05.md)
**User Story**: [US-020](../user-stories/US-020.md)
**ADR refs**: ninguno.

## Pre-condiciones

- Tiene al menos un ReviewReport propio.

## Flujo principal

1. Usuario pide `GET /me/reports`.
2. Sistema devuelve sus reports con `status`, `reason`, `resolution_note` si están resueltos.

## Post-condiciones

- Usuario ve seguimiento. Cierra el ciclo de confianza en la moderación.

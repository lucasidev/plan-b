# UC-013: Cargar historial manualmente

**Actor**: Alumno
**Epic**: [EPIC-03: Historial académico](../epics/EPIC-03.md)
**User Story**: [US-013](../user-stories/US-013.md)
**ADR refs**: [ADR-0004](../../decisions/0004-enrollment-guarda-hechos.md)

## Pre-condiciones

- Tiene StudentProfile.

## Flujo principal

1. Alumno selecciona Subject de su plan.
2. Elige `status` y `approval_method`.
3. Ingresa `grade` si corresponde.
4. Elige `term_id` (salvo equivalencia).
5. Sistema valida invariantes del EnrollmentRecord.
6. Sistema persiste el EnrollmentRecord.

## Post-condiciones

- Historial actualizado. Afecta estados derivados del grafo de carrera (disponibles, bloqueadas).

# UC-010: Registrarse

**Actor**: Visitante (que quiere convertirse en member)
**Epic**: [EPIC-02 — Identidad y autenticación](../epics/EPIC-02.md)
**User Story**: [US-010-b](../user-stories/US-010-b.md), [US-010-f](../user-stories/US-010-f.md)
**ADR refs**: —

## Pre-condiciones

- Ninguna.

## Flujo principal

1. Visitante envía email + password.
2. Sistema valida formato + unicidad del email.
3. Sistema hashea el password.
4. Sistema crea `User` con `role='member'`, `email_verified_at=NULL`.
5. Sistema envía email de verificación vía SMTP.

## Post-condiciones

- Cuenta creada pendiente de verificación. No puede publicar ni crear profiles hasta verificar.

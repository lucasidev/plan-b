# UC-034: Stats públicas agregadas

**Actor**: Visitor (sistema, no-auth)
**Epic**: [EPIC-01 — Catálogo público y exploración](../epics/EPIC-01.md)
**User Story**: [US-034](../user-stories/US-034.md)
**ADR refs**: [ADR-0009](../../decisions/0009-anonimato-como-regla-de-presentacion.md), [ADR-0034](../../decisions/0034-redis-como-cache-y-ephemeral-state.md)

## Pre-condiciones

- Ninguna. Endpoint público, sin auth.

## Flujo principal

1. Frontend (RSC en la landing `/auth`) llama a `GET /api/stats/public`.
2. Backend chequea cache Redis con key `stats:public`.
3. Cache hit: devuelve los counts cacheados.
4. Cache miss: ejecuta query Dapper cross-schema agregando counts de `Academic.Universities`, `Identity.Users` (role=member && !disabled), `Reviews.Reviews` (no soft-deleted). Guarda en Redis con TTL 1h.
5. Frontend hidrata el hero con los counts reales.

## Flujos alternativos

- **Endpoint falla (DB down, Redis down)**: frontend usa los valores hardcodeados de fallback que ya viven en el código. La landing no se rompe por una stat.

## Post-condiciones

- El visitor ve counts agregados en la landing.
- No se expone PII ni datos por universidad/carrera/usuario individual.

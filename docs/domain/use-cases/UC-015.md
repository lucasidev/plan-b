# UC-015: Editar una entrada del historial

**Actor**: Alumno
**Epic**: [EPIC-03 — Historial académico](../epics/EPIC-03.md)
**User Story**: [US-015](../user-stories/US-015.md)
**ADR refs**: [ADR-0004](../../decisions/0004-enrollment-guarda-hechos.md)

## Pre-condiciones

- Tiene un `EnrollmentRecord` propio.

## Flujo principal

1. Alumno abre la entrada.
2. Modifica `status`, `grade` o `approval_method`.
3. Sistema valida invariantes.
4. Sistema actualiza el EnrollmentRecord.

## Post-condiciones

- EnrollmentRecord actualizado.
- Puede invalidar reseñas asociadas si la edición la hace inconsistente (por ejemplo, vuelve a `cursando`).

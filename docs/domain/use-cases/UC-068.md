# UC-068: Deshabilitar cuenta member

**Actor**: Moderador o admin
**Epic**: [EPIC-02: Identidad y autenticación](../epics/EPIC-02.md), [EPIC-09: Backoffice de cuentas staff](../epics/EPIC-09.md)
**User Story**: [US-068](../user-stories/US-068.md)
**ADR refs**: [ADR-0009](../../decisions/0009-anonimato-como-regla-de-presentacion.md)

## Pre-condiciones

- Existe el User target con `role='member'`.

## Flujo principal

1. Actor setea `User.disabled_at=now()`, `disabled_reason`, `disabled_by=actor.id`.

## Post-condiciones

- Usuario no puede loguearse.
- Sus reseñas y datos quedan en DB (soft delete).
- El anonimato público se mantiene.

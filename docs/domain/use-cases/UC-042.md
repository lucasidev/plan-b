# UC-042: Navegar el área autenticada

**Actor**: Member logueado
**Epic**: [EPIC-04: Planificación de cuatrimestre](../epics/EPIC-04.md)
**User Story**: [US-042-f](../user-stories/US-042-f.md)
**ADR refs**: [ADR-0019](../../decisions/0019-single-nextjs-app-con-route-groups.md), [ADR-0023](../../decisions/0023-auth-flow-jwt-cookie-layout-guards.md)

## Pre-condiciones

- Member tiene sesión activa (cookies válidas).

## Flujo principal

1. Member entra a cualquier ruta dentro de `(member)`.
2. El layout server-side valida la sesión (guard existente).
3. Frontend renderiza AppShell con sidebar + topbar + área de contenido.
4. Member clickea un item del sidebar.
5. Si la ruta está implementada, navega.
6. Si la ruta es stub, ve un mensaje `Próximamente`.
7. Member abre el avatar dropdown abajo a la izquierda.
8. Selecciona "Cerrar sesión" → flujo de US-029-i.

## Flujos alternativos

- **Sesión inválida**: el guard del layout redirige a `/auth` antes de renderizar el shell.
- **Click fuera del avatar dropdown abierto**: el dropdown se cierra.

## Post-condiciones

- Member ve un chrome consistente en cualquier ruta autenticada.
- El sign-out es accesible desde cualquier vista.

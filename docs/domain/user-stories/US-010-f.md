## Como visitante, quiero una página de registro para crear mi cuenta desde la UI

Como visitante, quiero una página `/registro` con un form (email + password + confirm) que valide del lado cliente, llame al endpoint de backend, muestre errores legibles y me redirija a una pantalla post-success indicando que revise mi email, para tener flujo end-to-end sin tocar curl.

## Acceptance Criteria

- [x] Página `/registro` (App Router, RSC con cliente nested para form).
- [x] Form usa primitivas React 19 + TanStack Form (ADR-0022).
- [x] Validación cliente: formato email, password ≥ 12 chars, confirm matches.
- [x] Submit llama `POST /api/identity/register` via fetch wrapper.
- [x] Manejo de errores: 409 muestra "email ya registrado", 400 muestra mensaje del campo.
- [x] Post-success: redirige a `/verifica-tu-email` con instrucciones.
- [x] Estados: loading durante submit, disabled del botón.
- [x] Estilo con shadcn/ui (Card, Input, Button, Label).

## Sub-tasks

- [x] Crear página `/registro` con layout
- [x] Implementar form con TanStack Form
- [x] Wirear fetch wrapper con manejo de errores
- [x] Crear página `/verifica-tu-email` post-success
- [x] Estados loading/error/success en UI

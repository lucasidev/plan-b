## Como visitante, quiero un formulario de signup en la app para crear mi cuenta sin tocar curl

Como visitante, quiero `/sign-up` con form de email + password (+ confirm), validación cliente, llamada al backend de register existente, manejo de errores legible (409 "email ya registrado", 400 por campo) y al éxito redirección a la pantalla post-success que me indica revisar el correo.

Comparte el componente `AuthView` con `/sign-in` (mismas tabs, mismo split layout) per el mockup. La tab "Crear cuenta" del AuthView postea a este endpoint. Aterriza en el mismo PR que [US-028-f](US-028-f.md) (login frontend) porque comparten componente.

## Acceptance Criteria

- [ ] Página `(auth)/sign-up/page.tsx` renderiza `AuthView` con tab "Crear cuenta" activa.
- [ ] Form: email + password + confirm. Validación cliente con Zod (formato email, password ≥ 12, confirm matches).
- [ ] Submit dispara server action que llama `POST /api/identity/register`.
- [ ] 201 → redirect a `/sign-up/check-inbox?email=<email>` con instrucciones de revisar mail.
- [ ] 409 → muestra "Ya existe una cuenta con ese email" cerca del input.
- [ ] 400 → muestra el mensaje de validación del campo correspondiente.
- [ ] Estados de UI: loading durante submit, button disabled.
- [ ] Estilo usa primitivas del design system (`AuthSplit`, `Button`, `DisplayHeading`, `Lede`).
- [ ] Tab "Ingresar" navega a `/sign-in` (Next `<Link>` con prefetch).
- [ ] Copy en español rioplatense.

## Sub-tasks

- [ ] Componente `AuthView` shared con tabs internos signup/signin (compartido con US-028-f)
- [ ] Página `(auth)/sign-up/page.tsx`
- [ ] Página `(auth)/sign-up/check-inbox/page.tsx`
- [ ] Server action `signUpAction` que llama el backend
- [ ] Form con React 19 primitives + Zod (per ADR-0022 — 1-3 fields, no necesita TanStack Form)
- [ ] Estados loading/error/success en UI

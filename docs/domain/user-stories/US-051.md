## Como moderator, quiero resolver un report como upheld o dismissed para limpiar la cola

Como moderator, quiero dos endpoints (uphold / dismiss) con resolutionNote: uphold cascadea a otros reports open de la misma review y la pone en removed; dismiss puede devolver la review a published si era el único open.

## Acceptance Criteria

- [ ] `POST /api/admin/reports/{id}/uphold` con `{ resolutionNote }` → setea report status=upheld + cascade upheld a otros reports abiertos de la misma review + Review.status=removed + audit log.
- [ ] `POST /api/admin/reports/{id}/dismiss` con `{ resolutionNote }` → setea report status=dismissed; si era el único open y review estaba under_review → vuelve a published.

## Sub-tasks

- [ ] Comando `UpholdReportCommand` + handler con cascade
- [ ] Comando `DismissReportCommand` + handler
- [ ] Endpoints Carter
- [ ] Audit log entries
- [ ] Cross-BC events vía Wolverine outbox (ADR-0030)
- [ ] UI moderator

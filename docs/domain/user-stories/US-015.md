## Como alumno, quiero modificar una entrada de mi historial para corregir datos o actualizar status

Como alumno, quiero un endpoint PATCH que modifique una EnrollmentRecord propia (corregir datos, actualizar status de "cursando" a "aprobada" al terminar el cuatrimestre), re-validando invariantes y, si la edición invalida una Review existente, emitir un evento que la mande a `under_review` (ADR-0032).

## Acceptance Criteria

- [ ] `PATCH /api/me/enrollment-records/{id}` con cambios.
- [ ] Valida ownership.
- [ ] Re-valida invariantes post-edit.
- [ ] Si edit destructive (cambio a status='cursando' habiendo Review existente): UI confirma + emite `EnrollmentRecordEdited` que Reviews observa para invalidar la review (pasa a `under_review`).

## Sub-tasks

- [ ] Comando `UpdateEnrollmentCommand` + handler
- [ ] Endpoint Carter PATCH
- [ ] Detección de "destructive edit" (status downgrade)
- [ ] Integration event `EnrollmentRecordEdited`
- [ ] Handler en Reviews que invalide la review

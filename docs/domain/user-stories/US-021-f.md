## Como user pendiente de verificar, quiero un botón "reenviar email" en la página post-registro

Como user que se registró pero no recibió el email (o le venció el token), quiero un botón visible en la pantalla post-registro o en `/verifica-tu-email` que llame al endpoint de reenvío y me confirme que se envió, con feedback claro si quedo en cooldown por rate limit, para no quedar atorado.

## Acceptance Criteria

- [ ] Página `/verifica-tu-email` con botón "Reenviar email de verificación".
- [ ] Click dispara server action que llama `POST /api/identity/resend-verification`.
- [ ] Estados de UI: idle, loading, sent (toast "Te lo enviamos de nuevo, revisá tu bandeja"), rate-limited (toast "Esperá unos minutos, ya enviamos varios").
- [ ] Anti-enumeration: el botón se muestra siempre que el user esté autenticado-pero-no-verificado, sin diferenciar si el email es válido (aunque para un user logueado siempre lo es).
- [ ] Mensajes en español rioplatense.

## Sub-tasks

- [ ] Page Next.js `/verifica-tu-email` (App Router server component)
- [ ] Botón con shadcn/ui Button + estado loading
- [ ] Server action que llama el endpoint backend
- [ ] Toast de feedback (sent / rate-limited / error genérico)
- [ ] Test e2e con Playwright (opcional, post-MVP)

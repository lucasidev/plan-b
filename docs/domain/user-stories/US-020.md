## Como user que reportó, quiero ver el estado de mis reports para cerrar el ciclo de confianza con la moderación

Como user que reportó reseñas, quiero un endpoint que liste mis reports con su status y resolution_note (si fue resuelto), para tener feedback de qué pasó con mis flags.

## Acceptance Criteria

### Backend

- [ ] `GET /api/me/reports` lista los reports del user logueado.
- [ ] Cada item incluye: `id`, `reviewId`, `reason`, `details`, `status`, `resolutionNote` (si resuelto), `createdAt`, `resolvedAt`.
- [ ] No expone identidad del autor de la review reportada (anonimato sigue valiendo, ADR-0009).
- [ ] Paginación por `createdAt DESC`.
- [ ] Solo reports propios (filtro por `reporter_id = user.id`).

### Frontend

- [ ] Lista en `/cuenta/mis-reportes` con badge de status (open / upheld / dismissed) y `resolution_note` cuando corresponde.

## Sub-tasks

- [ ] Read model Dapper
- [ ] Endpoint Carter
- [ ] UI lista en perfil
- [ ] Integration test: user solo ve sus reports, no de otros

## Notas de implementación

- **Cerrar el loop de confianza**: si el user reporta y nunca sabe qué pasó, percibe que la moderación no hace nada. Mostrar el outcome (upheld con nota, dismissed con nota) le da agency y confianza en el sistema.
- **Anonimato del autor sigue valiendo**: el reporter no necesita saber quién escribió la review reportada. La identidad solo se expone a moderators (US-053).

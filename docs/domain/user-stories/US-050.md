## Como moderator, quiero ver cola de reseñas en revisión ordenada por urgencia para priorizar mi trabajo

Como moderator, quiero un endpoint que liste las reseñas en status='under_review', ordenadas por count(reports open) DESC y antigüedad ASC, para atacar primero las urgentes.

## Acceptance Criteria

### Backend

- [ ] `GET /api/admin/reviews/queue` retorna reviews `status = 'under_review'` ordenadas por `COUNT(reports open) DESC`, luego `created_at ASC` (antigüedad).
- [ ] Cada item: review id, subject + teacher info, snippet del texto, count de reports open, fecha de paso a `under_review`, motivo (filter automático vs threshold de reports).
- [ ] Identidad del autor visible al moderator (no anonimizado). Es el actor que tiene legitimidad de verla por mandato de moderación.
- [ ] Paginación. Default 20.
- [ ] Requiere `role IN ('moderator', 'admin')`.
- [ ] Filtros opcionales en query: `?reason=` (motivo del primer report), `?olderThan=` (días, ej. `?olderThan=7`).
- [ ] Default sort: count(reports open) DESC, antigüedad ASC. Configurable con `?sort=oldest|newest|most_reported`.

### Frontend

- [ ] Cola moderator con badges por urgencia.

## Sub-tasks

- [ ] Read model Dapper

## Notas de implementación

- **Identidad visible solo a moderators**: ADR-0009. El anonimato es de presentación al público, no de storage. Moderator necesita ver quién es el autor para detectar abuso (mismo user reporteado N veces, etc.).
- **Orden por urgencia**: count de reports DESC prioriza casos con más demanda. Antigüedad ASC desempata: más viejos primero, evita que casos queden congelados al fondo.
- **Cross-BC**: Moderation owns el listing pero la query joinea Reviews + ReviewReports. Read model con Dapper cross-schema OK acá (lectura de presentación, no FK).

## Como sistema (scheduled job), quiero expirar registros no verificados después de 7 días

Como sistema, quiero un job daily que setee `expired_at = now()` en users con `email_verified_at IS NULL` y `registered_at < now() - 7 days`, para que el email vuelva a estar disponible (vía índice único parcial) y la cola de pending no crezca indefinidamente, sin acción del usuario.

## Acceptance Criteria

- [ ] Job daily a las 3 AM UTC (Wolverine scheduled o hosted service).
- [ ] Query: `User WHERE email_verified_at IS NULL AND expired_at IS NULL AND registered_at < now() - 7 days`.
- [ ] Por cada match: setea `expired_at = now()`. El email vuelve a estar disponible vía índice único parcial.
- [ ] Emite `UnverifiedRegistrationExpired` event por cada user expirado (Wolverine outbox).
- [ ] Logging estructurado: cantidad procesada por run + IDs (en debug).
- [ ] Integration test: insertar 3 users (uno old non-verified, uno old verified, uno reciente non-verified), correr el job, verificar que solo el primero quedó expirado.

## Sub-tasks

- [ ] Comando `ExpireUnverifiedRegistrationsCommand` + handler que ejecuta la query
- [ ] Scheduled job (Wolverine `[ScheduledJob("0 3 * * *")]` o `IHostedService` con timer)
- [ ] Migration: índice único parcial sobre `email` para users `WHERE expired_at IS NULL`
- [ ] Logging estructurado con conteo
- [ ] Integration test del job

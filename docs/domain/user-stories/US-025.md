## Como alumno, quiero editar una simulación guardada para ajustar la combinación

Como alumno con SimulationDraft propia, quiero modificar la composición (subjectIds, label) sin tener que crearla de cero, manteniendo la visibility actual.

## Acceptance Criteria

### Backend

- [ ] `PATCH /api/me/simulations/drafts/{id}` con `{ subjectIds[]?, label? }`.
- [ ] Valida ownership.
- [ ] Re-valida que cada subject pertenezca al CareerPlan del profile.
- [ ] Re-valida correlativas (las materias seleccionadas deben ser todas `disponible` para el alumno).
- [ ] Mantiene visibility actual (no cambia `private`/`shared` automáticamente).
- [ ] `subject_ids` resultante no vacío.
- [ ] Authorization: solo el owner.
- [ ] Editar una simulación pública la mantiene pública (no se vuelve privada por editar).

### Frontend

- [ ] Edit form sobre la vista de detail del draft.

## Sub-tasks

- [ ] Comando `UpdateSimulationDraftCommand`
- [ ] Endpoint PATCH
- [ ] UI edit form

## Notas de implementación

- **Re-validar correlativas en cada edit**: el historial del alumno puede haber cambiado entre create y edit (cargó más materias). Una combinación que era válida puede haberse vuelto inválida o viceversa. El handler usa el mismo domain service de US-016.
- **No flip automático a private si era shared**: editar una simulación shared no la oculta del corpus público. Si el alumno quiere "esta versión no es lo que quería compartir", primero unshare, después edit, después share. Coherente con la idea de que share es decisión explícita.

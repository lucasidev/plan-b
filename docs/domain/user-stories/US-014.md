# US-014: Importar historial desde PDF/texto

**Effort**: L

## Como alumno avanzado, quiero subir mi historial en PDF o texto para no cargar 30 materias a mano

Como alumno con muchas materias cursadas, quiero subir un PDF (o pegar texto) con mi historial, que un worker async lo procese, resuelva materias contra el catálogo via códigos, cree los EnrollmentRecord y me reporte qué pudo resolver y qué quedó pendiente.

## Acceptance Criteria

### Backend

- [ ] `POST /api/me/historial-imports` con multipart (PDF) o body texto + `{ sourceType }` (`pdf` | `text`).
- [ ] Crea `HistorialImport` con `raw_payload` JSONB (output del parser) y `status = 'pending'`.
- [ ] Worker async (Wolverine handler) procesa: extrae materias, status, grades del raw_payload.
- [ ] Resuelve contra `Subject.code` del CareerPlan del profile.
- [ ] Por cada match: crea `EnrollmentRecord`. Si ya existía uno con mismo `(student, subject, term)`, respeta el existente y registra la colisión en results (no sobrescribe).
- [ ] Reporte de no-resueltos persiste en `HistorialImport.results` (parte del raw_payload, JSONB).
- [ ] Cliente puede consultar status: `GET /api/me/historial-imports/{id}` con `{ status, error?, parsed_at?, results }`.
- [ ] Si el parser falla, `status = 'failed'`, `error = <msg>`. El alumno reintenta o cae al flujo manual (US-013).
- [ ] Tamaño máximo del upload: 5 MB (request body). Excedido devuelve 413 `enrollments.import.payload_too_large`.
- [ ] PDFs protegidos con password: rechazar con 422 `enrollments.import.encrypted` y mensaje 'Subí el PDF sin contraseña' al user.
- [ ] Multi-page: soportado. El parser concatena texto de todas las páginas antes de parsear.
- [ ] Worker timeout: 60s por import. Si excede, marca `HistorialImport.status='failed'` con `errorReason='timeout'`. El user puede reintentar.

### Frontend

- [ ] Página de upload + status polling. Por ahora US backend-only.

## Sub-tasks

- [ ] Aggregate `HistorialImport` con `raw_payload` JSONB
- [ ] Endpoint upload (multipart o body texto)
- [ ] Endpoint status `GET /api/me/historial-imports/{id}`
- [ ] Worker Wolverine handler que procesa pending → parsed/failed
- [ ] Parser PDF: PdfPig 0.1.10+ (open-source, .NET nativo, suficiente para texto plano de PDFs académicos).
- [ ] Resolver de materias contra `Subject.code`
- [ ] Integration tests: PDF válido con 30 materias, PDF inválido, texto pegado con typos en códigos, colisión con EnrollmentRecord existente

## Notas de implementación

- **JSONB para raw_payload**: ADR-0006. El shape del payload varía según `source_type` (PDF parseado vs texto pegado), JSONB lo absorbe sin migrations cada vez que mejoramos el parser. El reprocesado es trivial: rehacemos las columnas tipadas desde el JSONB.
- **Async via Wolverine**: el upload responde 202 con el ID; el alumno hace polling al status. Procesar inline bloquearía el request por segundos (PDF parsing).
- **Conflict resolution**: si ya existe un `EnrollmentRecord` para `(student, subject, term)`, el import lo respeta (no sobrescribe). El reporte muestra "MAT201 ya estaba cargada manualmente, salteada". Evita destruir trabajo previo del alumno.
- **Materias no resueltas**: el reporte deja claro qué códigos no se pudieron mapear. El alumno puede agregarlas manualmente con UC-013. No fallamos el import por no-resoluciones parciales.

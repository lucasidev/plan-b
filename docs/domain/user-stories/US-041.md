## Como docente autor de la response, quiero editar mi respuesta para corregir o ampliar

Como autor de una TeacherResponse, quiero editar el texto, sólo el author original puede editarlo.

## Acceptance Criteria

### Backend

- [ ] `PATCH /api/reviews/{id}/teacher-response` con `{ text }`.
- [ ] Solo el `TeacherResponse.teacher_id` matching el TeacherProfile del user puede editar.
- [ ] El TeacherProfile debe seguir verified al momento del edit.
- [ ] `text` no vacío.
- [ ] Setea `updated_at = now()`.
- [ ] Emite `TeacherResponseEdited` integration event.
- [ ] Audit log entry con diff `{ before, after }`.
- [ ] Cooldown: máximo 3 edits por response en 24h. Excedido devuelve 429.

### Frontend

- [ ] Edit inline en la response.

## Sub-tasks

- [ ] Comando `UpdateTeacherResponseCommand`
- [ ] Endpoint PATCH
- [ ] UI edit
- [ ] Integration tests: docente author edita, docente no-author rechazado, docente sin verified vigente rechazado

## Notas de implementación

- **Authorization revalida verified al momento del edit**: si un admin de-verificó el TeacherProfile post-publicación de la response, el edit debe rechazarse. Mantiene la regla "solo verificados pueden actuar".
- **Sin re-run de filter sobre la response**: a diferencia de Review (US-018), la response no pasa por content filter. El docente está identificado con su nombre real, hay accountability. Si abusa, va por moderation manual.

# US-045: Mi carrera shell + 5 tabs (consolidación de vistas académicas)

**Status**: Sprint actual (S2), splitada en [US-045-a](US-045-a.md) + [US-045-b](US-045-b.md) + [US-045-c](US-045-c.md) + [US-045-d](US-045-d.md) + [US-045-e](US-045-e.md)
**Sprint**: S2
**Epic**: [EPIC-03: Historial académico](../epics/EPIC-03.md)
**Priority**: High
**Effort**: L (S por slice)
**ADR refs**: [ADR-0041](../../decisions/0041-rediseño-ux-post-claude-design.md)

> Esta US se splitea en 5 slices secuenciales (frontend-only, partidos por tab): `-a` shell + nav, `-b` tab Plan, `-c` tab Correlativas, `-d` tab Materias + Docentes, `-e` tab Historial. El shell entra primero, los tabs aterrizan en orden de prioridad visual. Cada slice cierra los AC del shell o de uno (o dos) tabs específicos.

## Como member, quiero ver toda mi carrera (plan, correlativas, materias, docentes, historial) en una sola pantalla con tabs para no saltar entre 5 entradas distintas del sidebar

Hoy las 4 vistas v1 (Plan, Materias, Docentes, Historial) viven en rutas separadas en el sidebar como stubs `ComingSoon`. La sesión de claude-design del 2026-05-02 zanjó consolidarlas en una ruta única `/mi-carrera` con 5 tabs. Razones: las 5 son del mismo dominio académico personal, dispersarlas infla nav sin payoff, y consolidar reduce 4-5 entradas del sidebar a 1.

## Acceptance Criteria

- [ ] Ruta `/mi-carrera` (route group `(member)`) con sub-tabs como query param `?tab=...`. Tabs:
  1. `?tab=plan`: Plan de estudios (heatmap por año/cuatrimestre, materias aprobadas / cursando / pendientes / bloqueadas).
  2. `?tab=correlativas`: Grafo SVG de correlativas con highlight por materia.
  3. `?tab=catalogo`: Lista de materias del plan con buscador local + filtros (año, cuatri, estado).
  4. `?tab=docentes`: Lista de docentes que dictan materias del plan, con búsqueda.
  5. `?tab=historial`: Tabla del historial académico (entradas cargadas) + CTA para cargar / importar.
- [ ] **Default tab = `plan`** si no hay query param.
- [ ] El tab actual queda en URL (sharable + back/forward del browser navega entre tabs).
- [ ] Sidebar v2: borrar entradas viejas (`/plan`, `/subjects`, `/professors`, `/history`) y dejar solo `/mi-carrera`.
- [ ] **Drawers / detalles** (sub-rutas):
  - `/mi-carrera/materia/[code]`: drawer con stats de la materia + comisiones + reseñas.
  - `/mi-carrera/docente/[id]`: drawer con resumen colectivo + reseñas con respuesta.
- [ ] **Stub data en cada tab** hasta que aterrice backend de Academic CRUD (US-060 a US-065). Cada bloque con TODO claro de qué endpoint reemplaza el mock.

## Sub-tasks

- [ ] `app/(member)/mi-carrera/page.tsx`: server component con tabs.
- [ ] `app/(member)/mi-carrera/materia/[code]/page.tsx` + `docente/[id]/page.tsx` (drawers como modal o nested).
- [ ] `features/mi-carrera/components/{plan-grid,plan-graph,subject-list,teacher-list,history-table}.tsx`.
- [ ] `features/mi-carrera/data/` con mocks alineados al canvas v2 (cada array con TODO).
- [ ] Borrar stubs viejos `app/(member)/{plan,subjects,professors,history}/page.tsx`.
- [ ] Actualizar `Sidebar` para reemplazar las 4 entradas por una sola "Mi carrera".
- [ ] Tests de componentes para cada tab (smoke + a11y básica).

## Notas de implementación

- **Backend deuda diferida**: el ADR-0041 marca esto explícito. Mi carrera v2 aterriza con stub data. Cuando aterrice Academic CRUD (US-060..065) cada tab reemplaza su mock por fetcher real con TanStack Query (RSC prefetch + HydrationBoundary, patrón estándar del frontend).
- **Cuándo se cierra el backend**: lo decidís vos según cómo venga S2. Si el sprint da el respiro, el backend mínimo de plan + correlativas + catálogo entra en S2. Si no, queda para S3 antes que Planificar.
- **Heatmap del plan**: el componente PlanGrid del bundle v1 (`canvas-mocks/v2-screens-2.jsx`) es la referencia visual. Color por estado (aprobada / cursando / pendiente / bloqueada).
- **Grafo de correlativas**: SVG simple a mano (no library). Nodos = materias, aristas = "requiere".
- **Historial**: entrada lista de la tabla → drawer de edición (cuando aterrice US-013/US-015 cargar/editar manual).

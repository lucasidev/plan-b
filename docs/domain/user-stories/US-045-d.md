## Como member, quiero buscar materias y docentes de mi plan en listas filtrables y abrir drawers con su detalle para encontrar info sin saltar entre 5 pantallas

Cuarto slice del rebuild de Mi carrera. Cubre 2 tabs (Materias y Docentes) en un solo slice porque comparten queries del módulo Academic (catalog) + reseñas, y los drawers cierran el loop visual de los tabs anteriores (Plan + Correlativas linkean al drawer de materia).

## Acceptance Criteria

### Tab Catalogo (Materias)

- [ ] Tab `?tab=catalogo` renderea `features/mi-carrera/components/subject-list.tsx`.
- [ ] Lista de materias del plan, ordenadas por año + cuatrimestre por defecto.
- [ ] **Buscador local** (input filter) que matchea por nombre o código.
- [ ] **Filtros**: por año (1°, 2°, ...), por cuatri (1°C, 2°C, anual), por estado (aprobada, cursando, disponible, bloqueada).
- [ ] Cada item de la lista: nombre + código + año/cuatri + chip de estado + CTA "Ver detalle" → drawer.
- [ ] **Empty state**: si filtros no matchean nada, "No encontramos materias con esos filtros."

### Tab Docentes

- [ ] Tab `?tab=docentes` renderea `features/mi-carrera/components/teacher-list.tsx`.
- [ ] Lista de docentes que dictan materias del plan (deducidos del mock).
- [ ] **Buscador local** por nombre.
- [ ] Cada item: nombre + materias que dicta + CTA "Ver detalle" → drawer.

### Drawers

- [ ] **Drawer de materia** `app/(member)/mi-carrera/materia/[code]/page.tsx`. Open via URL directo (sharable) o como modal sobre la lista (preferred UX). MVP: como página dedicada que renderea un layout drawer-style (panel a la derecha, lista atenuada a la izquierda).
- [ ] Contenido del drawer materia:
  - Nombre + código + año/cuatri + estado.
  - Correlativas (lista clickeable que abre la otra materia).
  - Comisiones disponibles (lista mock).
  - Reseñas top 3 con CTA "Ver todas" (link a `/reseñas?subjectId=...` cuando aterrice US-005).
  - CTA "Cerrar" → vuelve a la lista (`router.back()`).
- [ ] **Drawer de docente** `app/(member)/mi-carrera/docente/[id]/page.tsx`.
- [ ] Contenido del drawer docente:
  - Nombre + foto (placeholder si no hay).
  - Materias que dicta + comisiones.
  - Resumen colectivo (rating promedio, número de reseñas).
  - Reseñas con respuesta destacadas.
  - CTA "Ver todas las reseñas" + "Cerrar".

### Mock data

- [ ] `features/mi-carrera/data/{teachers.ts,reviews.ts,comisiones.ts}`. Cada array con TODOs de los endpoints reales (US-061, US-005, US-024).

### Linking cross-tab

- [ ] Click en celda del Plan (US-045-b) → drawer de materia.
- [ ] Click en nodo del grafo Correlativas (US-045-c) → drawer de materia.
- [ ] Click en correlativa de un drawer materia → otro drawer materia (navegación entre detalles).

## Out of scope

- Datos reales: 100% mock. TODOs apuntan a endpoints futuros.
- Crear / editar reseñas desde el drawer: CTA "Escribir reseña" si la lista de reseñas está vacía, pero solo navega a una página stub. Implementación queda en US-017.
- Filtros avanzados (rango de notas, ordenar por rating): MVP simple.
- Lazy loading / paginación: las listas son chicas (≤ 50 items del plan). Render todo de una.
- Drawer como modal sobre la lista (sin navegar): MVP usa página dedicada por simplicidad. Si se siente saltón, refactor a modal con `parallel routes` queda como deuda.

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| Búsqueda matchea 0 materias | Empty state. |
| Filtro estado=bloqueada con todas las correlativas cumplidas | Lista vacía + empty state. |
| Click en CTA "Ver detalle" + URL inválido `/materia/INEXISTENTE` | 404 page del route group. Mensaje "Esa materia no existe en tu plan." + link "Volver a Materias". |
| Drawer de materia con 0 reseñas | "Aún no hay reseñas. Sé el primero" + CTA stub. |
| Drawer de docente que no dicta ninguna materia del plan (caso patológico) | 404 con mensaje "Ese docente no dicta en tu plan." |
| Click en correlativa A desde drawer de materia B | Navega a `/mi-carrera/materia/A`. Browser back vuelve al drawer de B. |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** mock con 20 materias, **when** Lucía entra a `?tab=catalogo`, **then** ve la lista completa ordenada por año.
2. **Given** Lucía escribe "Programación" en el buscador, **when** termina de tipear, **then** la lista filtra a las materias con "Programación" en nombre.
3. **Given** Lucía clickea "Ver detalle" en una materia, **when** la página carga, **then** se ve el drawer con info + reseñas + CTA cerrar.
4. **Given** Lucía está en el drawer de Materia X y clickea una correlativa Y, **when** la página carga, **then** se ve el drawer de Y. Browser back vuelve a X.
5. **Given** Lucía está en `?tab=docentes`, **when** clickea "Ver detalle" de un docente, **then** se ve el drawer con sus materias + reseñas.

## Notas de implementación

- **Drawer como página dedicada vs modal**: optamos por página dedicada (`/mi-carrera/materia/[code]`) por simplicidad y porque es sharable. Visualmente se trata como drawer (panel derecho) pero la URL es real. Cuando aterrice un patrón de parallel routes (`@modal`), se evalúa migrar.
- **Filter state en URL vs local**: MVP usa `useState` local (no persiste filtros en URL). Si crece la necesidad, se migra a `nuqs` para sharable filters.
- **Mock de teachers deducido del mock plan**: `teachers.ts` lista los docentes inventados que dictan las materias del plan inventado. Mantener consistencia: cada teacher_id aparece en `comisiones.ts`.
- **Reviews mock**: 3-5 reseñas inventadas por materia, suficientes para mostrar el "top 3" del drawer.
- **Linking cross-tab a drawers**: los hrefs son `/mi-carrera/materia/${code}` o `/mi-carrera/docente/${id}`. El usuario al cerrar el drawer hace `router.back()` que lo devuelve a donde estaba (Plan, Correlativas, Catalogo, etc.).
- **Performance**: lista < 50 items, filter local instantáneo. No hace falta debounce.

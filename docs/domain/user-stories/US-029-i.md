## Como user autenticado, quiero un botón "Cerrar sesión" que limpie mi sesión completamente

Como user logueado, quiero clickear "Cerrar sesión" en el menu de cuenta y que mi refresh token se revoque, mis cookies se limpien y termine en `/sign-in`. Si en otra tab ya cerré sesión, este botón es idempotente — no rompe.

Slice integrated: backend + frontend en un solo PR porque es chico (un endpoint trivial + un button).

## Acceptance Criteria

- [ ] `POST /api/identity/sign-out`. No requiere body.
- [ ] Backend extrae refresh token del cookie `planb_refresh`, hashea, llama `DEL identity:refresh:{hash}` en Redis.
- [ ] Backend setea `Set-Cookie` con `planb_session` y `planb_refresh` `Expires=Thu, 01 Jan 1970 00:00:00 GMT` para limpiar.
- [ ] 204 sin body.
- [ ] Idempotente: sin cookies o con cookies ya inválidas → 204 igual.
- [ ] Frontend: botón "Cerrar sesión" en el dropdown del avatar (sidebar component, cuando aterrice). Mientras tanto, link en `/dashboard` placeholder.
- [ ] Click → `signOutAction` server action → `POST /api/identity/sign-out` → `redirect('/sign-in')`.
- [ ] Integration test: login con Lucía + sign-out + intentar refresh con la cookie vieja → 401.

## Sub-tasks

- [ ] Application: `SignOutCommand` + handler que invalida refresh en Redis
- [ ] Endpoint Carter `SignOutEndpoint` que limpia cookies
- [ ] Server action `signOutAction` en `features/identity/actions.ts`
- [ ] Botón en frontend (placeholder en dashboard hasta que exista sidebar real)
- [ ] Integration test: sign-out + intentar refresh → 401

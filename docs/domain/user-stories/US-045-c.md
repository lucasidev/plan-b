## Como member, quiero ver el grafo de correlativas de mi plan para entender qué me bloquea para anotarme a otra materia

Tercer slice del rebuild de Mi carrera. Reemplaza el stub `correlativas-stub.tsx` con un grafo SVG de las dependencias entre materias.

## Acceptance Criteria

- [ ] Tab `?tab=correlativas` renderea `features/mi-carrera/components/correlativas-graph.tsx`.
- [ ] **Grafo SVG** con materias como nodos circulares + aristas dirigidas (flecha de "requiere" → "destino").
- [ ] **Nodos coloreados por estado** del alumno (mismos códigos visuales que el tab Plan: verde aprobada, azul cursando, gris disponible, gris oscuro bloqueada).
- [ ] **Click en un nodo** highlightea las materias que lo requieren (descendientes) + las que requiere (ancestros). Resto del grafo se atenúa (`opacity-30`).
- [ ] **Hover en un nodo** muestra tooltip con nombre completo + estado.
- [ ] **Layout del grafo**: simple, hand-coded (sin library externa). Layered layout: año 1 arriba, año N abajo. Posición x dentro de la fila por orden del cuatri.
- [ ] **Mock data** en `features/mi-carrera/data/correlativas.ts`: array de aristas `{ from, to }` + reuso del `plan.ts` para los nodos. TODO de qué endpoint reemplaza el mock.
- [ ] **Empty state**: si el plan no tiene correlativas declaradas, placeholder "Tu plan no tiene correlativas modeladas todavía."
- [ ] **Leyenda visual**: pie del grafo con colores + significado, igual que el tab Plan.
- [ ] **Click en un nodo bloqueado** abre el mismo drawer de materia (US-045-d) con las correlativas pendientes destacadas.

## Out of scope

- Library de grafos (vis.js, cytoscape, d3-force): SVG hand-coded para mantener el bundle chico y el visual controlado.
- Drag de nodos / pan / zoom: layout estático en MVP. Cuando los planes sean grandes, se evalúa.
- Animaciones de transición al hacer click: transición CSS simple (`transition-opacity`), sin keyframes complejos.
- Edición del grafo (admin): out of scope.
- Responsividad mobile: degraded acceptable per ADR-0041 (web-first).

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| Materia sin correlativas (root del DAG) | Nodo aislado en su fila. Click no highlightea nada (no hay ancestros ni descendientes). |
| Materia con correlativa cíclica (caso patológico, no debería existir en planes reales) | El layout falla seguro: detecta ciclo, log warning, renderea sin las aristas problemáticas. |
| Plan con > 30 nodos | El SVG se renderea pero puede ser denso. MVP: zoom out con CSS scale. Si en uso real es ilegible, pasa a deuda. |
| Click en nodo cuando ya hay otro highlighteado | Resetea highlight + aplica al nuevo. |
| Click fuera del grafo | Resetea highlight. Todos los nodos vuelven a opacidad 100%. |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** mock con plan + correlativas, **when** la home renderea el tab Correlativas, **then** se ve un grafo con nodos por materia + aristas.
2. **Given** un nodo en estado "aprobada", **when** se inspecciona, **then** color verde.
3. **Given** un nodo bloqueado por correlativas, **when** click, **then** los ancestros (correlativas pendientes) se highlight en color destacado.
4. **Given** un nodo aislado (sin correlativas), **when** click, **then** el resto se atenúa pero ningún otro nodo se highlightea.
5. **Given** click fuera del grafo, **when** se inspecciona, **then** todos los nodos están en opacidad 100% (highlight reseteado).

### Cobertura por capa

- **Component / vitest + RTL**: `correlativas-graph.test.tsx` con mock injectado: verifica render de nodos + aristas + highlight on click + reset.
- **Unit / vitest**: `getAncestors(nodeId, edges)` y `getDescendants(nodeId, edges)` aparte, testeo exhaustivo con DAG inventado.
- **E2E Playwright**: cubierto en spec único de cierre de US-045 (cuando los 5 tabs estén).

## Notas de implementación

- **Layered layout simple**: agrupar nodos por `year`, ordenar por `term` dentro del año, asignar x equiespaciado y y por año. Sin algoritmos sofisticados.
- **Aristas como líneas**: `<line>` con `stroke` + marker de flecha al final. Para evitar crossings horribles cuando hay muchas aristas, no se hace optimización: layered layout es suficiente para planes típicos.
- **Reactivity del highlight**: `useState` para el `highlightedNodeId`. El grafo recalcula opacidades en cada render. Cheap porque los nodos son pocos (< 50).
- **Click outside del SVG**: handler en el `<svg>` con `onClick` que resetea el state. Click en `<circle>` hace `e.stopPropagation()` para que no se resetee.
- **Tooltips**: shadcn Tooltip o `<title>` SVG nativo (sin libs).
- **Reuso visual con tab Plan**: mismos colores. Si los tokens están en design system, importarlos. Si no, definir constantes locales en `lib/colors.ts` del feature.

## Como alumno, quiero publicar una simulación al corpus público anonimizada para que otros alumnos vean opciones consideradas

Como alumno, quiero un endpoint que cambie la visibility de mi draft a 'shared' (anonimizado), aparezca en el corpus público filtrable por plan + cuatrimestre, y poder volverla privada cuando quiera.

## Acceptance Criteria

### Backend

- [ ] `POST /api/me/simulations/drafts/{id}/share` flips visibility a `shared`, setea `shared_at = now()`.
- [ ] `POST /api/me/simulations/drafts/{id}/unshare` vuelve a `private`, limpia `shared_at`.
- [ ] Valida ownership del draft.
- [ ] Mientras es shared, aparece en `GET /api/simulations/public?planId=&termId=` anonimizada (sin `owner_profile_id`).
- [ ] `GET /api/simulations/public` requiere user logueado con `StudentProfile` (privacy: el corpus público es para alumnos, no para visitantes anónimos).

### Frontend

- [ ] Toggle "Compartir con la comunidad" en cada draft. Tooltip explica que se anonimiza.

## Sub-tasks

- [ ] Comandos `ShareSimulationCommand` / `UnshareSimulationCommand`
- [ ] Endpoints Carter
- [ ] UI toggle compartir

## Notas de implementación

- **Anonimato en public listing**: el endpoint público nunca expone `owner_profile_id`. Misma lógica que reviews (ADR-0009). Identidad existe en DB para el dueño (puede unshare), pero no se filtra externamente.
- **Premium feature**: ADR-0028. Compartir requiere haber reseñado al menos N veces (regla a definir). MVP: abierto. Cuando aterrice el sistema de premium, gate ahí.
- **Constraint del data-model**: `visibility = 'shared'` requiere `shared_at NOT NULL`. El handler setea ambos atómicamente; el toggle a private limpia `shared_at`.

## Como visitante, quiero registrarme con email + password (backend) para crear mi cuenta

Como visitante, quiero un endpoint que acepte mi email y password, valide los datos, hashee la password, persista la User con role=member, genere un VerificationToken, envíe el email de verificación vía SMTP y responda 201, para tener mi cuenta creada esperando verificación.

## Acceptance Criteria

- [x] `POST /api/identity/register` acepta `{ email, password }`.
- [x] Email validado: formato + unicidad case-insensitive entre users non-expired.
- [x] Password ≥ 12 chars, hasheado con BCrypt cost 12.
- [x] Crea User con role=member, email_verified_at=null.
- [x] Genera VerificationToken (TTL 24h, opaque 256-bit base64url, purpose=UserEmailVerification).
- [x] Envía email vía SMTP (Mailpit en dev, real en prod) con link a `/verify-email?token=...`.
- [x] Retorna 201 con `{ id, email }`.
- [x] Retorna 409 si email duplicado, 400 si validación.
- [x] Integration tests cubren happy path + duplicate + invalid input.

## Sub-tasks

- [x] Comando `RegisterUserCommand` + handler en módulo Identity
- [x] Endpoint Carter `RegisterEndpoint`
- [x] Servicio de hashing BCrypt
- [x] VerificationToken como child entity de User (ADR-0033)
- [x] SmtpEmailSender con configuración de host/port
- [x] Integration tests con WebApplicationFactory + Postgres real + Mailpit

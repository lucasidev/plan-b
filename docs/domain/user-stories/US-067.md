# US-067: Crear cuentas staff

**Status**: Backlog
**Sprint**: —
**Epic**: [EPIC-09 — Backoffice de cuentas staff](../epics/EPIC-09.md)
**Priority**: Medium
**Effort**: S
**UC**: [UC-067](../use-cases/UC-067.md)
**ADR refs**: ADR-0008

## Como admin, quiero crear cuentas con role moderator/admin/university_staff para incorporar gente al equipo operativo

Como admin, quiero un endpoint que cree Users con role staff, email auto-verified (no hay flow de auto-registro para estos roles, ADR-0008).

## Acceptance Criteria

### Backend

- [ ] `POST /api/admin/staff-users` con `{ email, password, role }` (role: `moderator` | `admin` | `university_staff`).
- [ ] Email se persiste con `email_verified_at = now()` (auto-verified).
- [ ] No hay flow de auto-registro para estos roles ([ADR-0008](../../decisions/0008-roles-exclusivos-profiles-como-capacidades.md)). El endpoint de `/sign-up` rechaza role staff.
- [ ] Password hasheado con BCrypt cost 12 (mismo que registro normal).
- [ ] Si role = `university_staff`, requiere `universityId` para scopear el dashboard.
- [ ] No puede crear `StudentProfile` ni `TeacherProfile` para users staff (invariante).
- [ ] Requiere `role = 'admin'`.

### Frontend (deuda diferida)

- [ ] UI admin para staff onboarding con copy claro sobre permisos.

## Sub-tasks

- [ ] Comando `CreateStaffUserCommand`
- [ ] Endpoint Carter
- [ ] Authorization admin only
- [ ] UI admin
- [ ] Integration tests: admin crea moderator, admin crea university_staff con universityId, member intenta crear staff rechazado, role inválido rechazado

## Notas de implementación

- **No auto-registro para staff**: ADR-0008. Los roles `moderator`, `admin`, `university_staff` son administrativos. La incorporación es manual y trazable. Evita escalation de privilege por self-service.
- **Email auto-verified**: el admin que crea es responsable de verificar la identidad fuera de banda. No tiene sentido mandar mail de confirmación a una cuenta operativa.
- **`university_staff` requiere scope**: el dashboard institucional (US-080) solo muestra data de la university scopeada. Crear un staff sin scope sería inseguro (default access-all).

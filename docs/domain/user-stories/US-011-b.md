## Como user recién registrado, quiero un endpoint que consuma mi token de verificación para activar mi cuenta

Como user recién registrado, quiero que el sistema acepte el token del link de verificación, lo valide (no expirado, no consumido), marque mi `EmailVerifiedAt = now()` y responda 204 (o redirect a UI de confirmación), de forma idempotente: doble click no rompe.

## Acceptance Criteria

- [ ] `GET /api/identity/verify?token=<value>` consume el VerificationToken con purpose=UserEmailVerification.
- [ ] Idempotente: doble click no rompe (segundo retorna 200 sin cambio).
- [ ] Token inválido / expirado / consumido → 400 con error code distinguible.
- [ ] Marca `User.EmailVerifiedAt = now()`.
- [ ] Retorna 204 (o redirect a UI de confirmación según ADR de UX).
- [ ] Como parte del slice se refactoriza `EmailVerificationToken` → `VerificationToken` child entity de User (ADR-0033).
- [ ] Integration tests cubren: happy path, doble click, token expirado, token inválido.

## Sub-tasks

- [ ] Refactor `EmailVerificationToken` → `VerificationToken` con discriminator `Purpose`
- [ ] Comando `VerifyEmailCommand` + handler
- [ ] Endpoint Carter `VerifyEmailEndpoint`
- [ ] Validación de TTL y consumed_at
- [ ] Integration tests para happy path + edge cases (doble click, expirado, inválido)

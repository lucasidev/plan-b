## Como member con claim pendiente sin email institucional, quiero subir evidencia para que un admin verifique manualmente

Como member con claim pendiente sin email institucional disponible, quiero subir archivos (DNI, contrato, etc.) que queden persistidos como evidencia, dejando el claim en cola del admin para revisión manual.

## Acceptance Criteria

### Backend

- [ ] `POST /api/me/teacher-claims/{id}/evidence` multipart con uno o más archivos (PDF, JPG, PNG).
- [ ] Valida ownership del claim.
- [ ] Persiste referencias a archivos vía abstraction `IFileStorage` (local FS en dev, S3-compatible en prod).
- [ ] Claim queda en estado de "pending manual review" (visible en cola de admin).
- [ ] Tamaño máximo configurable, default 10 MB por archivo.
- [ ] Tipos MIME whitelist: `application/pdf`, `image/jpeg`, `image/png`.
- [ ] Audit log con quién subió, qué archivos, cuándo.
- [ ] `IFileStorage` interface en `Planb.SharedKernel.Abstractions.Storage`, implementación `MinioFileStorage` usando paquete `Minio` (oficial, .NET).
- [ ] Cantidad máxima de archivos por request: 5. Excedido devuelve 400 `claim.evidence.too_many_files`.
- [ ] Tamaño máximo total: 20 MB. Excedido devuelve 413.
- [ ] Retención: archivos vinculados a un claim se borran 30 días después de approve/reject. Job mensual via Wolverine ScheduledJob limpia los huérfanos.

### Frontend

- [ ] Form de upload en la página de claim.

## Sub-tasks

- [ ] Endpoint upload multipart
- [ ] Storage abstraction (`IFileStorage`: local en dev, S3 en prod)
- [ ] Setear claim en estado "pending manual review"
- [ ] UI upload form
- [ ] Integration test: archivo válido subido, archivo demasiado grande rechazado, MIME no permitido rechazado

## Notas de implementación

- **`IFileStorage` como puerto, no S3 directo**: persistence ignorance (ADR-0017) aplica también para storage de archivos. El dominio no sabe si es local o S3. Permite tests con storage en memoria y dev sin AWS.
- **Soporte de MIME whitelist**: evitar payloads ejecutables o scripts subidos como "evidencia". Hay scanners más sofisticados pero el MVP cubre el 90% con whitelist + tamaño.
- **Sin OCR ni parsing automático**: la evidencia la lee el admin, no el sistema. Mantener simple. Si en el futuro hay volumen alto, considerar tooling.
- **MinIO ganó sobre AWS S3 / R2** por la regla de no-paid services del proyecto. En dev se levanta como container en docker-compose; en prod self-hosted en el mismo servidor.

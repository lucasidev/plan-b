## Como user que clickea el link de verificación, quiero ver una pantalla de confirmación legible

Como user recién registrado que clickea el link recibido por email, quiero llegar a `/verify-email?token=...`, que la página llame al endpoint de verify y muestre un estado claro: éxito (con CTA a login), token inválido o expirado (con CTA a re-enviar email), para no quedarme con dudas de qué pasó.

## Acceptance Criteria

- [ ] Página `/verify-email` lee `token` del query string (RSC server component).
- [ ] Server-side: llama al endpoint `GET /api/identity/verify?token=...` y resuelve estado.
- [ ] Estado success: muestra confirmación + CTA "Iniciar sesión".
- [ ] Estado error (token inválido/expirado/consumido): muestra mensaje + CTA "Reenviar verificación".
- [ ] Sin token en URL: redirige a `/registro`.
- [ ] Estilo coherente con shadcn/ui.

## Sub-tasks

- [ ] Crear página `/verify-email` con server component
- [ ] Llamada server-side al endpoint backend
- [ ] Estados visuales: success / error / no-token
- [ ] CTA a login y a reenviar verificación

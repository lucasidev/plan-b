## Como user que clickea el link de verificación, quiero ver una pantalla de confirmación legible

Como user recién registrado que clickea el link recibido por email, quiero llegar a `/verify-email?token=...`, que la página llame al endpoint de verify y muestre un estado claro: éxito (con CTA a login), token inválido o expirado (con CTA a re-enviar email), para no quedarme con dudas de qué pasó.

## Acceptance Criteria

- [x] Página `/verify-email` lee `token` del query string (RSC server component).
- [x] Server-side: llama al endpoint `POST /api/identity/verify-email` con body `{ token }` y resuelve estado.
- [x] Estado success: muestra confirmación + CTA "Iniciar sesión".
- [x] Estado error (token inválido/expirado/consumido/invalidated): muestra mensaje específico por tipo + CTA workaround "Registrarme de nuevo".
- [x] Estado already_consumed: tratado como cuasi-éxito (la cuenta está verificada, link reusado), CTA directo a "Iniciar sesión".
- [x] Sin token en URL: muestra estado `missing_token` con CTA workaround.
- [x] Estilo coherente con primitivas del design system (`AuthSplit`, `Button`, `DisplayHeading`, `Lede`).

## Sub-tasks

- [x] Crear página `/verify-email` con server component
- [x] Llamada server-side al endpoint backend
- [x] Estados visuales: success / already_consumed / invalid / expired / invalidated / missing_token / unknown
- [x] CTA a login y workaround "registrate de nuevo"

## Notas de implementación

- **Endpoint real**: el AC original mencionaba `GET /api/identity/verify?token=...`, pero el endpoint backend implementado es `POST /api/identity/verify-email` con body `{ token }`. Se ajustó el AC para reflejar la realidad.
- **"Reenviar verificación" no se implementó como botón funcional** porque el endpoint backend de resend (US-021) todavía no existe. El workaround es un CTA "Registrarme de nuevo" que lleva a `/sign-up`: registrarse otra vez con el mismo email regenera el token de verificación. Cuando US-021-f aterrice, ese CTA se reemplaza por un resend real sin romper la estructura del componente.

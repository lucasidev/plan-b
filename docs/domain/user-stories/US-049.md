# US-049: Editor de reseña 6 campos numerados con preview vivo

**Status**: Backlog
**Sprint**: candidato a S4
**Epic**: [EPIC-05 — Sistema de reseñas](../epics/EPIC-05.md)
**Priority**: High
**Effort**: L
**ADR refs**: [ADR-0041](../../decisions/0041-rediseño-ux-post-claude-design.md)

## Como member que terminó de cursar, quiero un editor estructurado en 6 pasos numerados con preview en vivo para escribir una reseña completa sin perderme y viendo cómo va a verse

La sesión de claude-design del 2026-05-02 zanjó el editor con 6 campos numerados (no formulario plano) + sidebar con preview vivo. El editor cubre la creación; el editar (US-018) reusa el mismo componente.

## Acceptance Criteria

- [ ] Ruta `/reseñas/escribir/[cursadaId]` (route group `(member)`).
- [ ] **Layout**: dos columnas. Izquierda = formulario 6 campos numerados. Derecha = preview vivo de la review tal como va a verse en el feed.
- [ ] **Campo 1 — Rating general (1-5 ★)**: requerido. Stars clickeables.
- [ ] **Campo 2 — Dificultad (1-5)**: requerido. Slider o stars distinto color.
- [ ] **Campo 3 — Horas de estudio/semana**: opcional. Slider 0-30.
- [ ] **Campo 4 — Texto libre**: opcional, sin mínimo. Textarea con counter (max 4000 chars).
- [ ] **Campo 5 — Tags rápidos**: chip selector múltiple. Set inicial (ejemplo, no aprobado): claro, exige, responde tarde, TPs buenos, aprueba justo, finales-flexibles, parciales-justos, exámenes-orales, copyleft-friendly. Set definitivo TBD en otra US.
- [ ] **Campo 6 — Recomendaciones sí/no** (2 toggles):
  - "Recomendarías esta cursada".
  - "Volverías a tomar a este docente".
- [ ] **Header del editor**: nombre de materia + docente + comisión + cuatri (data del cursada).
- [ ] **Preview vivo** se actualiza on change (no debounced, react render normal).
- [ ] **Botón "Publicar"** disponible solo si campos 1, 2 y 6 completos (los obligatorios).
- [ ] **Botón "Guardar borrador"**: opcional para esta US si complica scope. Decidir en planning.
- [ ] **Cancelar** vuelve a `/reseñas?tab=pendientes`.
- [ ] Después de publicar, redirige a `/reseñas?tab=mias` con la nueva reseña visible.

## Sub-tasks

### Backend

- [ ] `POST /api/reviews` con `{ cursadaId, rating, difficulty, hoursPerWeek?, text?, tags[], wouldRecommendCourse, wouldRetakeTeacher }`.
- [ ] Validación: rating 1-5, difficulty 1-5, hoursPerWeek 0-30, tags from allowed set, una sola review por cursadaId del mismo user (constraint unique).
- [ ] Tests: happy path, duplicado (409), valores fuera de rango (400 ProblemDetails RFC 7807).

### Frontend

- [ ] `app/(member)/reseñas/escribir/[cursadaId]/page.tsx` (server component que prefetch la cursada).
- [ ] `features/write-review/{api.ts,actions.ts,schema.ts (zod),types.ts,components/{review-editor,preview-card,star-input,difficulty-slider,hours-slider,tags-picker,toggle-recommendations}.tsx}`.
- [ ] TanStack Form con Zod schema compartido (4+ campos = complejo, ADR-0022).
- [ ] Hook `usePreview` que mapea estado del form a la card de preview.
- [ ] Tests de componentes (cada input, validation, submit happy path).

## Notas de implementación

- **6 campos numerados, no formulario plano**: el numerado da progreso visible. Ayuda con la fricción de "es mucho relleno".
- **Preview vivo**: el alumno ve cómo va a aparecer en el feed mientras escribe. Reduce el "publicar y darse cuenta que quedó feo".
- **Tags TBD**: el set inicial es ejemplo. Decidir taxonomy concreta en otra US o discusión separada. Hardcodear el set en una constante por ahora.
- **Una review por cursada**: enforced por unique constraint backend. Si el alumno ya tiene review para esa cursada, redirect al editar (US-018) en vez de crear.
- **Preview reusable**: el componente `<ReviewCard>` que se usa en feed (US-048) y en preview es el mismo. Single source of truth visual.

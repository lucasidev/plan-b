# US-077-f: Panel de notificaciones (dropdown anclado al bell del topbar)

**Status**: Backlog
**Sprint**: candidato a S5+
**Epic**: [EPIC-05: Sistema de reseñas](../epics/EPIC-05.md) (notificaciones del corpus social)
**Priority**: Medium
**Effort**: M (frontend); backend en [US-077-b](US-077-b.md) splitada en 3 sub-slices ([b-1](US-077-b-1.md) core / [b-2](US-077-b-2.md) subscribers / [b-3](US-077-b-3.md) email)
**ADR refs**: [ADR-0040](../../decisions/0040-notifications-como-bounded-context.md), [ADR-0041](../../decisions/0041-rediseño-ux-post-claude-design.md)

> Esta US es solo el frontend del panel. El backend de Notifications vive en [US-077-b](US-077-b.md), splitada en 3 sub-slices ([b-1](US-077-b-1.md) core + read API + mutations / [b-2](US-077-b-2.md) subscribers Wolverine cross-BC / [b-3](US-077-b-3.md) email delivery con SMTP genérico). **Hard dep en b-1** para dejar de usar mock; b-2 y b-3 son soft deps (sin ellas el sistema funciona con notifs cargadas manualmente / solo canal in-app).

## Como member, quiero un panel de notificaciones desde el bell del topbar que me liste lo que pasó con mis reseñas, lo que respondieron docentes, lo que cambió en mi plan, y los recordatorios, para no perderme avisos sin tener que navegar manualmente

Sección ⑩ del canvas v2 (`v2-screens-5.jsx::V2Notif`). El panel es un dropdown anclado al `V2Bell` del topbar (componente `v2-shell.jsx`). Presenta notificaciones agrupadas por recencia (Hoy / Esta semana / Antes), con tabs Todas / No leídas, mark-as-read individual + bulk, y CTA "Ver todas las notificaciones".

## Acceptance Criteria

- [ ] **Trigger**: click en el `<V2Bell/>` del topbar (que ya existe en `v2-shell.jsx::V2Bell`).
- [ ] **Estado del panel**:
  - Cerrado por default.
  - Abierto: dropdown con backdrop semi-transparente + click outside cierra.
  - `Esc` cierra.
- [ ] **Layout**: anchor al bell del topbar. Width 420px. Position absolute below the bell.
- [ ] **Header del panel**:
  - Heading "Notificaciones" + count "{N} sin leer".
  - Tabs: `Todas` | `No leídas`.
  - Botón secundario "Marcar todas como leídas" (disabled si N=0).
- [ ] **Items agrupados por recencia**:
  - **Hoy**: items con `createdAt` en últimas 24h.
  - **Esta semana**: items con `createdAt` últimos 7 días (no Hoy).
  - **Antes**: el resto.
- [ ] **Item type schema** (en `features/notifications/types.ts`):
  ```ts
  type NotificationKind =
    | 'review_response'        // docente respondió tu reseña
    | 'review_useful'          // alguien marcó tu reseña como útil
    | 'review_new_in_subject'  // nueva reseña en materia que seguís
    | 'plan_period_open'       // se abrió la inscripción del próximo cuatri
    | 'plan_change'            // tu plan cambió (docente nuevo, comisión cerrada)
    | 'system';                // mensaje del equipo plan-b (raro)

  type Notification = {
    id: string;
    kind: NotificationKind;
    title: string;          // "Brandt respondió tu reseña"
    body?: string;          // copy contextual corto
    actorAvatar?: string;   // initials del docente / system
    targetUrl: string;      // adónde navega al click
    createdAt: string;      // ISO
    readAt: string | null;
  };
  ```
- [ ] **Renderer por kind**: cada kind tiene un copy template + icono. Mantener el formato visual del canvas.
- [ ] **Click en item**:
  - Marca como leída (`readAt = now()`).
  - Navega a `targetUrl`.
  - Cierra el panel.
- [ ] **Mark-as-read individual**: botón secundario al hover sobre el item (icono ✓).
- [ ] **Mark-all-as-read**: button del header.
- [ ] **Empty state** (`v2-notif-empty`): copy "Estás al día" + ilustración minimalista (sin ilustraciones, fontmono char `✓` grande).
- [ ] **Footer del panel**: link mono "Ver todas las notificaciones →" navega a `/notificaciones` (página completa).
- [ ] **Bell badge**: count de no leídas hasta 9, `9+` si más. Hide si 0.
- [ ] **Mock data tipada** en `features/notifications/data/notifications.ts`:
  - Array de 8 mocks (mix de los 6 kinds, mix de leídas/no leídas).
  - TODO claro de qué endpoint reemplaza.

## Out of scope

- **Backend real de notificaciones**: cubierto por [US-077-b](US-077-b.md) y sub-slices. Esta US frontend consume los endpoints expuestos por [US-077-b-1](US-077-b-1.md).
- **Push notifications nativas / web push**: out (post-MVP).
- **Email summary de notificaciones**: cubierto parcialmente por US-072 (Ajustes notificaciones), pero la implementación es backend separada.
- **Página `/notificaciones` completa**: el footer linkea ahí pero la página completa es US separada (mismo modelo de data, vista lista paginada).
- **Filtros más finos por kind**: out de MVP. Solo Todas / No leídas.
- **Real-time push del bell badge**: en MVP el badge se actualiza al fetch (polling 30s o on-focus). Real-time WebSocket es post-MVP.

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| 0 notificaciones | Empty state visible. Bell sin badge. |
| Más de 9 no leídas | Badge muestra `9+`. |
| Click en item con `targetUrl` rota o que requiere permisos | Navega normal. Si la página destino devuelve 404/403, eso es responsabilidad de US-078-f (errores). |
| Notificación de hace 60 días | Cae en grupo "Antes" sin agrupación adicional. |
| Mock con `kind` desconocido | Fallback a icono genérico + copy "Algo pasó en plan-b". Log warning en consola. |
| Tab `No leídas` con 0 ítems | Muestra "Estás al día". |
| User abre el panel mientras hay nuevas llegando | Se hace refetch al abrir (no live). |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** Lucía con 3 notificaciones no leídas, **when** entra a `/home`, **then** el bell muestra badge `3`.
2. **Given** Lucía clickea el bell, **when** el panel se abre, **then** ve heading "Notificaciones" + tabs + items agrupados.
3. **Given** panel abierto en tab `No leídas`, **when** clickea "Marcar todas como leídas", **then** todos los items se marcan + badge desaparece.
4. **Given** panel abierto, **when** Lucía clickea un item de tipo `review_response`, **then** se marca como leído + navega a `/reseñas?tab=mias` (o el targetUrl del mock) + panel se cierra.
5. **Given** Lucía sin notificaciones, **when** abre el panel, **then** ve empty state.
6. **Given** panel abierto, **when** presiona `Esc`, **then** se cierra.

## Notas de implementación

- **Por qué `(member)` y no `(public)`**: notificaciones son personales, requieren sesión.
- **Polling vs WebSocket**: en MVP polling 30s alcanza. Cuando el corpus social crezca y haga falta real-time, se evalúa WebSocket o SSE (probablemente Wolverine push).
- **Bell badge ya está renderizado en `V2Shell::V2Bell` con prop `count`**: cuando se port a la app, el count viene del hook `useNotifications().unreadCount`.
- **Los `targetUrl` se resuelven en backend** (la notificación trae el link concreto). Frontend solo navega.
- **Testing offline**: el panel es read-only para el user; en offline (US-076-f) se sigue pudiendo abrir y ver, solo el "marcar como leído" queda disabled.

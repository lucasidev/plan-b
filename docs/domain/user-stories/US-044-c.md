## Como member, quiero ver pendientes contextuales (reseñar lo que cursé + planificar lo que viene + movimientos del entorno) para no perder hilo de qué falta

Tercer y último slice del rebuild de `/home`. Inserta los 3 bloques de la columna derecha del mock `V2Inicio`: "Reseñá lo que cursaste" (acent, con materias cerradas pendientes) + "Pensando en lo que viene" (CTA Planificar) + "Movimientos" (feed de notifs).

## Out of scope

- Datos reales (Reviews + Notif): 100% mock. TODOs marcan endpoints reales.
- Dismiss / mark-as-read de movements: items son display-only.
- Filtros / sort de movements: orden hardcoded.
- Real-time de movements (websocket / polling): snapshot al render. Cuando aterrice US-Notif, evaluamos.
- Mobile diseñado.
- Confetti / animaciones: descartados.

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| Mock to-review vacío | Empty state "Cuando cierres una materia te avisamos para reseñarla." Bloque sigue visible (mantiene grilla). |
| Mock movements vacío | Empty state "Sin novedades por ahora." Bloque sigue visible. |
| `note=null` en una materia pendiente de reseñar | El subtítulo `{prof} · nota {note}` muestra `{prof} · sin nota` o omite la nota. |
| Body de movement muy largo | Wrap en 2-3 líneas máximo. line-height 1.4 mantiene legibilidad. |
| Click en "Reseñar →" cuando US-017 no aterrizó | Link a stub con copy "Disponible pronto" + "Volver a Inicio". |
| Click en "Planificar 2027 →" cuando US-016 no aterrizó | Mismo patrón: stub. |
| Mock con > 5 movements | Render todos (no truncate en MVP). Si crece, paginación. |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** mock to-review con 3 materias, **when** la home renderea, **then** el bloque "Reseñá lo que cursaste" muestra 3 cards con CTA "Reseñar →".
2. **Given** mock to-review vacío, **when** la home renderea, **then** se ve el empty state.
3. **Given** click en "Planificar 2027 →", **when** se inspecciona, **then** redirige a stub o se loggea TODO.
4. **Given** mock movements con 3 items, **when** se inspecciona, **then** los 3 aparecen con timestamp izquierda y body a la derecha.
5. **Given** un item de movement con body largo, **when** se inspecciona, **then** wrap a múltiples líneas sin romper layout.
6. **Given** la home completa con `-a` + `-b` + `-c`, **when** Lucía entra, **then** ve el header + período + columna izq (En curso, Más adelante) + columna der (Reseñá, Planificar, Movimientos).

## Notas de implementación

- **`{nextYear}` en "Pensando en lo que viene"**: hardcoded a `currentPeriod.year + 1`. Cuando aterrice un módulo de calendario académico, se enchufa.
- **`closed` en pendingReview**: ya viene formateado como string ("2025·2c"). Cuando el backend lo expone, llega como `{ year: 2025, half: '2c' }` y se formatea acá.
- **Movements timestamps en mock pre-formateados**: `"hace 2h"`, `"hace 1d"`, `"hace 3d"`. Cuando aterrice notif backend, llegan como ISO 8601 y se formatean con `formatRelativeTime(date, now)` (helper a crear en `lib/time.ts` o reusar uno existente).
- **Card acent del bloque "Reseñá"**: el border-color usa OKLCH directo (`oklch(0.85 0.07 55)`). Si no calza con tokens existentes, agregar una variable `--color-accent-border` en `globals.css`. MVP: hardcode el OKLCH.
- **CTAs con TODO**: cada uno con comment `// TODO(US-017): reseña backend + UI` o similar. Hacer obvio el destino futuro.

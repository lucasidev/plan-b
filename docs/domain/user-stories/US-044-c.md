## Como member, quiero ver pendientes (reseñas que tengo que escribir, completar historial) + actividad reciente para no perder hilo de qué falta y qué pasó

Tercer y último slice del rebuild de `/home`. Inserta los dos bloques restantes debajo de "Cursando ahora" (US-044-b). Cierra el rebuild de la home v2.

## Acceptance Criteria

- [ ] **Bloque "Pendientes"** (`features/dashboard-home/components/pending-list.tsx`):
  - Lista vertical de items pendientes para el alumno.
  - Tipos de pendiente del MVP:
    1. "Reseñá `<materia>` con `<docente>`" (cuando aterrice US-017): item con CTA → `/reseñas/escribir?subjectId=...&teacherId=...`.
    2. "Completá tu historial" (si el flag de `historial cargado` está en `false`): CTA → `/onboarding/welcome` o `/mi-carrera/historial` cuando aterrice US-013.
    3. Otros tipos quedan abiertos para US futuras (ej. "Confirmá tu mail si no lo hiciste" cuando aterrice resend).
  - Cada item con icono + texto + CTA secundario. Si la lista está vacía: empty state "Estás al día. Volvé cuando aparezca algo nuevo."
- [ ] **Bloque "Actividad reciente"** (`features/dashboard-home/components/recent-activity.tsx`):
  - Lista vertical de últimas notifs / eventos del entorno.
  - Tipos del MVP (todos mock):
    1. "`<docente>` respondió tu reseña" (cuando aterrice US-024).
    2. "Subió la nota promedio de `<materia>`" (insight cuando aterrice US-Insights).
    3. "Tu reseña en `<materia>` recibió 3 votos útiles" (cuando aterrice US-040).
  - Cada item con timestamp relativo (ej. "hace 2h", "ayer"). Si la lista está vacía: empty state "Sin novedades por ahora."
- [ ] **Mock data** en `features/dashboard-home/data/{pending.ts,recent-activity.ts}`. Cada array con TODOs explícitos del endpoint que reemplaza el mock.
- [ ] Ambos bloques se renderean siempre (no condicionales por estado dominante). El layout v2 los ubica al lado / debajo de "Cursando ahora" según viewport.
- [ ] Layout responsive: en desktop, los 3 bloques (Cursando, Pendientes, Recientes) se ubican en 2 columnas (Cursando ocupa 2/3, los otros 2 apilados a la derecha en 1/3). En tablet / mobile, todo apilado vertical (web-first per ADR-0041, mobile como degraded).

## Out of scope

- Datos reales: 100% mock. Los TODOs marcan dónde enchufar Reviews + Notif + Insights cuando aterricen.
- Dismiss / mark-as-read de items: los items son display-only en MVP. Click en CTA navega a la URL pero no actualiza estado del item.
- Filtros / sort de la lista de pendientes: orden hardcoded en mock.
- Real-time updates (websocket / polling): la lista es snapshot al render. Refresh al navegar.
- Mobile diseñado: degraded acceptable.

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| Mock de pendientes vacío | Empty state "Estás al día." |
| Mock de actividad reciente vacío | Empty state "Sin novedades por ahora." |
| Más de 5 items en pendientes | Render hasta 5, luego "Ver más" como link al área correspondiente (ej. `/reseñas/pendientes` cuando aterrice US-048-c). MVP: simple truncate con "+ N más" sin link funcional. |
| CTA de un pendiente apunta a una ruta que no existe | TODO en código + redirige a `/home` con toast "Próximamente". |
| Item de actividad con timestamp futuro (ej. mock mal) | Renderea con "hoy" como fallback, no rompe. |
| Viewport ≤ 768px | Stack vertical: Cursando, Pendientes, Recientes uno debajo del otro. |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** mock de pendientes con 2 items, **when** la home renderea, **then** se ven los 2 con su CTA visible.

## Notas de implementación

- **Mock data shape de pendientes**: `{ id, type, title, ctaLabel, ctaHref, icon }`. Type tagged: `'review' | 'historial' | 'verify-email' | ...` para que el componente decida icono.
- **Mock data shape de actividad reciente**: `{ id, type, title, timestamp, icon }`. Timestamp ISO; el componente formatea con `formatRelativeTime`.
- **Layout grid en `home/page.tsx`**: Tailwind grid 2 cols en `md+` con `col-span-2` para Cursando y `col-span-1 row-span-1` para Pendientes y Recientes apilados. En mobile, `grid-cols-1` simple.
- **Empty states no son condicionales de "ocultar bloque"**: el placeholder mantiene la grilla estable (cero salto de layout cuando hay datos vs no).
- **CTAs con TODO**: la mayoría apunta a rutas futuras. Hardcode + comment del US destino.
- **Sin animaciones**: render estático. Cuando aterrice US de notifs real-time, evaluamos polling / websocket.

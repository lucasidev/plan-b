## Como member, quiero ver mis materias en curso (con su progreso, próximo evento y asistencia) y las que arrancan más adelante para entender de un vistazo cómo va mi cuatri

Segundo slice del rebuild de `/home`. Inserta los dos bloques de la columna izquierda del mock `V2Inicio`: "En curso" (cards con progreso y stats) + "Más adelante" (compacto, materias del cuatri siguiente).

## Acceptance Criteria

- [ ] **Bloque "En curso"** (`features/home/components/current-subjects-card.tsx`, port de V2Inicio líneas 96-158):
  - Card estándar con eyebrow h2 "En curso" + small "{N} materias" + ghost button "Ver todas →".
  - Filas de materias con grid `1fr auto 100px`, gap 14, padding `12px 0`, border-top entre filas.
  - **Cada fila muestra**:
    - Left: code mono · `<ModPill mod={...} />` · "com {N}" mono. Debajo: nombre de la materia (14px, weight 500). Debajo: `{prof} · {next}` (11.5px, color `var(--color-ink-3)`).
    - Middle: nota parcial mono 18px (si existe) + label "nota parcial" abajo. Si no existe: "sin notas" en italic, color `var(--color-ink-4)`.
    - Right: labels mono `sem {week}/{weeks}` y `{attendance%}` arriba; `<ProgressBar value={week} max={weeks} tone={week/weeks > 0.8 ? 'warm' : 'neutral'} />` debajo.
- [ ] **Bloque "Más adelante"** (`features/home/components/upcoming-subjects-card.tsx`, port de V2Inicio líneas 161-185), condicional `if (futuras.length > 0)`:
  - Card estándar con h2 "Más adelante".
  - Filas compactas (padding `9px 0`, border-top entre filas):
    - Left: code mono · `<ModPill />` · nombre (13px).
    - Right: `{next}` (mono, 11.5px). Por ej. "arranca el 5 ago".
- [ ] **Helper reusable**: `features/home/components/mod-pill.tsx` (port de V2Mod del mock). Recibe `mod: '1c' | '2c' | 'anual' | '1s' | '2s' | 'bim1' | 'bim2' | 'bim3' | 'bim4'` y renderea pill con label corto.
- [ ] **Mock data tipada** en `features/home/data/active-subjects.ts`:
  ```ts
  export type ActiveSubject = {
    code: string;
    name: string;
    mod: 'anual' | '1c' | '2c' | '1s' | '2s' | 'bim1' | 'bim2' | 'bim3' | 'bim4';
    com: string; // 'A' | 'B' | ...
    prof: string; // apellido del docente
    diff: 1 | 2 | 3 | 4 | 5;

    attendance: number | null;

  // TODO: cuando aterrice US-013 (cargar historial) + US-016 (simular) + Academic catalog,
  // reemplazar por fetch a EnrollmentRecord + Subject + Commission.
  export const activeSubjects: ActiveSubject[] = [ ... 5 items del mock ... ];
  ```
- [ ] El mock activo replica los 5 items del mock (ISW302, INT302, MAT401, SEG302, QUI201). 4 con `week > 0` (cursando) + 1 con `week=0` (futura).
- [ ] El subtitle del header (`-a`) consume `activeSubjects.filter(s => s.week > 0).length` y `.length - cursando.length` para los counts. `-b` cambia la página para que el subtitle sea dinámico (en `-a` estaba hardcodeado).
- [ ] Los componentes se insertan en la columna izquierda del `<section>` reservado por `-a`.

## Out of scope

- Drawer / página de detalle de materia: el CTA "Ver todas →" navega a `/mi-carrera?tab=catalogo` (cuando aterrice US-045-a) o stub.
- Datos reales: 100% mock. TODOs marcan los endpoints reales.
- Dismiss / interaction de las rows (ej. click en row abre modal): row es display-only en MVP.
- Filtros / sort de las materias: orden hardcoded del mock.
- Real-time updates: snapshot al render.
- Mobile diseñado.

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| Mock activo vacío (sin cursando) | Bloque "En curso" muestra empty state ("Cuando empieces a cursar, vas a ver tus materias acá"). Subtitle del header refleja "0 materias cursando". |
| Mock futuras vacío | Bloque "Más adelante" no se renderea (condicional). |
| Materia con `week > weeks` (caso patológico, mock mal armado) | ProgressBar clampa a 100%. Label "sem {week}/{weeks}" muestra el valor real igual. |
| Materia con `attendance=null` | Label "{attendance%}" no se renderea (o muestra guion "-"). |
| Materia con `note=null` | Columna nota muestra "sin notas" en italic. |
| Más de 6 materias activas | Render todas (no truncate en MVP). Si la lista es larga, `<CardScroll>` o similar; deuda futura. |
| Click en "Ver todas →" | Navega a `/mi-carrera?tab=catalogo` con TODO si US-045-a aún no mergea. |
| Modalidad inválida en mock (ej. `mod='trimestral'`) | `<ModPill>` cae a fallback guion "-" o renderea label genérico. |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** mock activo con 4 cursando + 1 futura, **when** la home renderea, **then** "En curso" muestra 4 filas y "Más adelante" muestra 1.
2. **Given** una row con `note=8`, **when** se inspecciona, **then** muestra el 8 mono 18px + "nota parcial" abajo.
3. **Given** una row con `note=null`, **when** se inspecciona, **then** muestra "sin notas" en italic.
4. **Given** mock futuras vacío, **when** la home renderea, **then** el bloque "Más adelante" no aparece.
5. **Given** una row con `attendance=0.92`, **when** se inspecciona, **then** label muestra "92%".
6. **Given** una row con `week=8, weeks=16`, **when** se inspecciona, **then** label muestra "sem 8/16" y la barra está al 50%.
7. **Given** una row con `week/weeks > 0.8` (faltan pocas semanas), **when** se inspecciona, **then** la progress bar usa tone "warm".

## Notas de implementación

- **`<ModPill>` se exporta para reuso**: US-045 (Mi carrera) lo va a usar también. Tipos compartidos.
- **Tone de la progress bar**: `'warm'` cuando faltan pocas semanas (`week/weeks > 0.8`), `'neutral'` el resto. Lógica en el componente.
- **Filter `s.week > 0`** = cursando. **Filter `s.week === 0`** = futura. Misma lógica que el mock (V2Inicio líneas 47-48).
- **Sin lazy load**: lista chica (5-10 items max), render todo.
- **Empty state** se diseña simple: card con copy + ícono opcional. No bloquea el layout (mantiene grilla estable).

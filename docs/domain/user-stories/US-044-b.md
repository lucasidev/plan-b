## Como member, quiero ver "Cursando ahora" con info útil de cada materia (semana, próximos parciales, asistencia) para entender de un vistazo cómo va mi cuatri

Segundo slice del rebuild de `/home`. Inserta el primer bloque secundario debajo de la pregunta dominante (US-044-a). Reemplaza el componente `coursing-now.tsx` placeholder de S1 por uno con cards de las materias activas con datos accionables.

## Acceptance Criteria

- [ ] `features/dashboard-home/components/cursando-now-v2.tsx` con cards de materias del período activo. Cada card muestra:
  - Nombre de la materia + comisión.
  - Semana actual del cuatri (ej. "Semana 8/16").
  - Próximo evento relevante (parcial / TP / final, con fecha relativa: "en 3 días", "mañana").
  - Asistencia acumulada (porcentaje + threshold visual: verde si > 75%, ámbar si 60-75%, rojo si < 60%).
  - CTA "Ver detalle" → `/mi-carrera?tab=catalogo&materia=<code>` (cuando aterrice US-045-d).
- [ ] **Mock data** en `features/dashboard-home/data/cursando-now.ts`: array de 3-4 materias inventadas. Cada array con `// TODO(US-013 + US-016): reemplazar mock por data real de Enrollments + Academic`.
- [ ] El bloque renderea solo si `getDominantQuestion` devolvió "¿cómo va tu cuatri?" o "¿qué te anotás este cuatri?" (estados con cuatri activo o próximo). Si la pregunta es "Empecemos por cargar tu historial", no se muestra.
- [ ] **Empty state**: si el mock devuelve array vacío, el bloque muestra un placeholder "Cuando empieces a cursar, vas a ver tus materias acá" (no se oculta totalmente, para mantener la grilla visual estable).
- [ ] Se inserta en el `<section>` placeholder dejado por US-044-a. La página `home/page.tsx` importa y renderea el componente.
- [ ] **Borrar** `features/dashboard-home/components/coursing-now.tsx` (placeholder de S1) + `features/dashboard-home/data/mock-coursing.ts` viejo. Si hay tests asociados, se reescriben para `cursando-now-v2.tsx`.

## Out of scope

- Datos reales: este slice es 100% mock. Los TODOs marcan dónde enchufar Enrollments + Academic cuando aterricen US-013 y US-016.
- Drawer de detalle de materia: el CTA "Ver detalle" navega a `/mi-carrera?tab=catalogo&materia=<code>` que aterriza con US-045-d. Mientras tanto, deja el link como `<a href>` con TODO.
- Notificaciones push de "tu parcial es mañana": eso es US-Notif futura.
- Edición de la materia desde la card: cualquier mutation queda fuera, este es solo display.
- Indicadores avanzados (ej. promedio actual): solo lo descrito en AC. Lo demás se evalúa cuando haya feedback.

## Edge cases

| Caso | Comportamiento esperado |
|---|---|
| Mock array vacío | Empty state con placeholder. No se oculta el bloque. |
| Materia con asistencia null o sin parciales próximos | Card renderea con guion ("--") en los campos faltantes. No rompe el layout. |
| Más de 6 materias en el array | Grid se acomoda con scroll horizontal o wrap a múltiples filas. MVP: wrap simple. |
| Fecha relativa de evento "hace 2 días" (parcial pasado) | Se muestra como "Pasado: parcial 1" con tono neutral. No se oculta. |
| Asistencia exactamente en threshold (75%) | Cae en banda "verde" (≥ 75%). |

## Test scenarios

### Críticos (Given-When-Then)

1. **Given** mock con 3 materias activas, **when** la home renderea (usuario en estado "cursando"), **then** se ven 3 cards con nombre + semana + asistencia visible.
2. **Given** mock con array vacío, **when** la home renderea, **then** se ve el empty state placeholder, no se oculta el bloque.
3. **Given** estado "Empecemos por cargar tu historial", **when** la home renderea, **then** el bloque "Cursando ahora" no aparece.
4. **Given** una card con asistencia 80%, **when** se inspecciona el indicador visual, **then** está en banda verde.
5. **Given** una card con asistencia 65%, **when** se inspecciona, **then** está en banda ámbar.

### Cobertura por capa

- **Component / vitest + RTL**: `cursando-now-v2.test.tsx` con mock injectado. Verifica render de cards + empty state + indicadores de asistencia.
- **Unit / vitest**: si la lógica de "qué banda es la asistencia" sale a un helper, testearlo aparte.

## Notas de implementación

- **Mock data shape**: alineada al ERD de `EnrollmentRecord` (Enrollments BC) + `Subject` y `Commission` (Academic BC). Cuando aterrice el endpoint real, el shape no cambia, solo se reemplaza la fuente.
- **Lógica de "semana 8/16"**: simple, fecha de inicio del cuatri + diff actual / 7. Mock hardcodea inicio del cuatri.
- **Eventos próximos**: el mock lista `[{type, date, label}]` ordenado, el componente toma el primero con fecha futura más cercana. Cuando aterrice US-Notif o un módulo de calendario académico, se cambia el origen.
- **Asistencia**: porcentaje calculado en mock. Cuando aterrice tracking real (US futura), se reemplaza.
- **CTA "Ver detalle"**: link con TODO comment. Cuando US-045-d aterrice, se enchufa con la query `?tab=catalogo&materia=<code>`.
- **No tocar `dominant-question.ts`**: el helper ya devuelve el estado, este slice solo lo consume.

## Como alumno con StudentProfile, quiero navegar simulaciones publicadas por otros alumnos del mismo plan + cuatrimestre para ver qué combinaciones se están considerando

Como alumno, quiero un endpoint que liste simulaciones shared del mismo plan + cuatrimestre (anonimizadas) con composición + métricas agregadas, para tener inspiración antes de armar la mía.

## Acceptance Criteria

### Backend

- [ ] `GET /api/simulations/public?planId=&termId=` retorna simulaciones `shared` del mismo CareerPlan + AcademicTerm.
- [ ] Cada item: composición (`subjectIds[]` con metadata de cada subject), label opcional, métricas agregadas (carga horaria total, dificultad promedio).
- [ ] Sin `owner_profile_id` ni info del autor (anonimato).
- [ ] Requiere user authenticated + `StudentProfile` activo.
- [ ] Filtra solo shared del mismo plan (no leakea cross-carrera).
- [ ] Paginación con cursor por `shared_at DESC`.
- [ ] Listing paginado cursor-based, pageSize=20.
- [ ] Solo simulaciones con visibility='public'. Filtros opcionales por careerPlanId y término.

### Frontend

- [ ] Feed público con filtros por término.

## Sub-tasks

- [ ] Read model con Dapper
- [ ] Endpoint Carter
- [ ] Authorization policy: requires `StudentProfile` active
- [ ] UI feed público

## Notas de implementación

- **Por qué requiere StudentProfile y no solo authenticated**: el corpus público es para alumnos que comparten su plan/term. Un visitante curioso o un user sin profile no aporta ni consume sentido del feed. Gate por `StudentProfile` activo del mismo plan que el query.
- **Filtro por planId en query, no inferred**: alumnos con multiple StudentProfiles necesitan elegir bajo qué plan están explorando. El query param explícito evita ambigüedad.
- **Métricas calculadas server-side**: cada item viene con las métricas pre-calculadas. Cliente no recalcula. Ahorra round-trips si el feed tiene 50 items.

## Como moderator o admin, quiero deshabilitar una cuenta member para suspender el acceso ante abuso o pedido del usuario

Como moderator o admin, quiero un endpoint que setee `disabled_at`, `disabled_reason` y `disabled_by` sobre una User, impida el login mientras esté disabled y soft-flag sus reseñas en presentación, sin borrar nada.

## Acceptance Criteria

- [ ] `POST /api/admin/users/{id}/disable` con `{ reason }`.
- [ ] Requiere role moderator o admin.
- [ ] Setea `User.disabled_at`, `disabled_reason`, `disabled_by=actor.id`.
- [ ] User no puede loguearse mientras esté disabled.
- [ ] Sus reviews quedan en DB (soft delete); presentation hace soft-flag.
- [ ] Emite `UserDisabled` integration event.

## Sub-tasks

- [ ] Comando `DisableUserCommand` + handler
- [ ] Endpoint Carter `/api/admin/users/{id}/disable`
- [ ] Authorization policy moderator|admin
- [ ] Login pipeline rechaza users con disabled_at NOT NULL
- [ ] Integration event Wolverine
- [ ] Reviews proyección: hide content cuando user.disabled_at NOT NULL

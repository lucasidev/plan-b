# US-071: Búsqueda global (topbar dropdown con Meilisearch)

**Status**: Backlog
**Sprint**: candidato a S5
**Epic**: [EPIC-01 — Catálogo público y exploración](../epics/EPIC-01.md)
**Priority**: High
**Effort**: L
**ADR refs**: [ADR-0039](../../decisions/0039-meilisearch-como-motor-de-busqueda-global.md), [ADR-0041](../../decisions/0041-rediseno-ux-post-claude-design.md)

## Como member, quiero buscar docentes / materias / comisiones desde el topbar con autocomplete instantáneo para llegar a cualquier entidad sin navegar por menús

La sesión de claude-design del 2026-05-02 zanjó la búsqueda global como **dropdown desde el topbar** (no página dedicada), con resultados agrupados, highlight del query, y atajos de teclado (⌘K). Backend = Meilisearch (ADR-0039).

## Acceptance Criteria

- [ ] Input de búsqueda en topbar v2, central. Placeholder rotativo con sugerencias del estilo "buscar docente, materia o comisión".
- [ ] **⌘K / Ctrl+K** abre el dropdown desde cualquier ruta autenticada.
- [ ] **Esc** cierra.
- [ ] **Flechas** navegan resultados.
- [ ] **Enter** sobre un resultado navega al detalle (`/mi-carrera/docente/[id]`, `/mi-carrera/materia/[code]`, etc.).
- [ ] **Resultados agrupados** por tipo:
  - Recientes (búsquedas previas del alumno, persistidas en localStorage).
  - Docentes.
  - Materias.
  - Comisiones.
- [ ] **Highlight del query** en cada resultado (Meilisearch lo soporta nativo).
- [ ] **Debounced** a 150ms (request mientras tipea sin saturar).
- [ ] **Estado vacío**: sin query → mostrar "Recientes" si las hay; con 0 resultados → mensaje + sugerencia.

## Sub-tasks

### Backend

- [ ] Endpoint `GET /api/search?q=&types=` que delega a Meilisearch.
- [ ] Indexers Wolverine que escuchan eventos de Academic (TeacherCreated, SubjectCreated, CommissionCreated) y empujan al índice Meilisearch correspondiente. Pipeline definido en ADR-0039.
- [ ] Reindex job idempotente para repoblar desde cero (ej. tras restore de backup).
- [ ] Tests integration: búsqueda con highlight, paginación, filtros por tipo.

### Frontend

- [ ] `components/layout/global-search.tsx` con dialog (shadcn) anclado al topbar.
- [ ] `features/global-search/{api.ts,hooks/{use-global-search,use-recent-searches}.ts,components/{search-results-list,search-result-item}.tsx}`.
- [ ] Atajo de teclado registrado en root layout.
- [ ] LocalStorage para recientes (max 10).

## Notas de implementación

- **No es página, es dropdown**: el alumno busca para *navegar*, no para "ver resultados de búsqueda". El dropdown da flow rápido: tipear → ver → enter → llegar.
- **Meilisearch >> Postgres FTS**: ADR-0039 explica el porqué. Tipo-tolerante por default, highlight gratis, latencia <50ms con corpus chico.
- **Indexer pipeline**: cada BC publica eventos via Wolverine outbox. Un projector (en el host del API o como background worker) escucha y empuja a Meilisearch. Eso preserva persistence ignorance + outbox durability.
- **Recientes en localStorage**: privacidad — no enviamos las búsquedas al server. Es UX local del browser.
- **Sin modal full-screen**: el dropdown es contenido sobre el contenido. Mantiene contexto visual de la página actual.
